# backlog-workflow

Backlog work means processing stories one by one from the canonical backlog source.

For each item:

- Read fully and restate the ask clearly.
- Check current code, tests, docs, UX, and architecture.
- If current state conflicts with the ask, pause and show exact differences.
- Wait for approval to overwrite, update, or skip.
- Rewrite the story if needed.
- Add accurate acceptance criteria.
- Add required prerequisite and follow-on stories.
- Update backlog and confirm exact changes.
