# full-qa-audit

Full QA is the complete repo audit workflow.

It includes:

- Repo discovery and milestone planning.
- Backdated story generation and cleanup.
- Story validation and coverage audit.
- Traceability and verification audits.
- Deep-pass planning and phased execution.
- Required validation checks.
- Commit checkpoints when explicitly requested.
- Backup branch creation before squash/rebase work.

When triggered with `full qa`, run the next appropriate step in this full procedure, not an abbreviated QA pass.

If repo docs conflict with this procedure, pause and show the conflict before continuing.
